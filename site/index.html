Architecture

Three planes. Strictly separated.

The dashboard never talks to devices — it renders a digital twin. Telemetry flows up, live state sits in the middle, commands flow down. Conflating them is how smart homes become spaghetti; HomeOS never does.

01

Telemetry up

Devices publish readings over real MQTT. Retained messages mean the dashboard loads instantly with last-known state; Last Will gives offline detection for free. Ingest validates, enriches, and writes history.

homeos/tele/studio/climate/temp → 21.2
homeos/state/studio/lights · retained
02

Live state

The twin store is the single source of current truth. The realtime API pushes deltas over WebSocket — a snapshot on connect, then only what changed, coalesced per animation frame.

GET /ws → snapshot, then deltas
{"room":"studio","temp":21.4}
03

Commands down

Controls publish intent, never state. The actuator's state echo is the only confirmation — optimistic UI with an ack timeout, so the interface degrades honestly when a device is unreachable.

homeos/cmd/studio/lights {"on":true,"level":80}
state echo confirms in ~200 ms
